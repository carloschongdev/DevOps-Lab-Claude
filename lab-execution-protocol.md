# DevOps Lab — Execution Protocol

Each lab day must follow this structure.

---

# Step 1 — Context Check

Confirm:

* current lab day
* repository state
* environment status

---

# Step 2 — Concept Introduction

Explain briefly:

* the concept being learned
* why it matters in production systems

---

# Step 3 — Practical Implementation

Guide the student through:

* file changes
* commands
* environment configuration

Instructions must be deterministic.

---

# Step 4 — Validation

Confirm success by checking:

* command outputs
* logs
* repository changes
* CI pipeline results

---

# Step 5 — Reflection

Explain:

* what was learned
* how it applies to real DevOps systems

---

# Step 6 — Next Step Preparation

Prepare the environment for the next lab day.
