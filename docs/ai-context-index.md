# AI Context Index

This document provides a structured overview of the DevOps Lab Claude control repository.

Its purpose is to help AI assistants quickly understand:

* the purpose of the repository
* the role of each document
* the current state of the DevOps Lab
* where to find relevant information

This improves the quality of instructions generated by AI mentors.

---

# Repository Purpose

This repository is not the DevOps Lab itself.

Instead, it acts as the **control repository used to guide the DevOps Lab through AI mentoring**.

The repository contains:

• mentoring prompts
• execution rules
• architectural documentation
• learning roadmap
• lab progress tracking

AI assistants must use this repository as the **source of truth** when generating instructions.

---

# Key Directories

## mentor/

Contains the configuration used to control the behavior of the AI mentor.

Files include:

claude-devops-mentor-prompt.md
Defines the role and behavior of the DevOps mentor.

lab-execution-protocol.md
Defines how lab instructions must be generated.

lab-rules.md
Defines operational rules that the lab must follow.

# AI Mentor Guidelines

AI assistants interacting with this repository must:

1. Review the repository structure before generating instructions.
2. Use the documentation in /docs as architectural reference.
3. Follow the rules defined in /mentor.
4. Avoid making assumptions about the state of the DevOps Lab.
5. Request file content before proposing modifications.
