# DevOps Lab Skill Matrix

This document defines the technical skill progression expected throughout the DevOps Lab.

The purpose of the skill matrix is to ensure that the laboratory develops a balanced set of DevOps engineering competencies.

Claude must consult this matrix when generating lab tasks.

The laboratory should not focus on a single tool but should develop a combination of:

- infrastructure understanding
- automation
- containerization
- CI/CD
- observability
- debugging
- system architecture thinking

---

# Skill Categories

The DevOps Lab focuses on the following core skill areas:

1. Linux & System Fundamentals
2. Git & Version Control
3. Containers (Docker)
4. CI/CD Pipelines
5. Infrastructure Concepts
6. Kubernetes & Orchestration
7. Observability & Monitoring
8. Debugging & Troubleshooting
9. Automation & Scripting
10. DevOps Architecture Thinking

Each skill must progressively develop across the laboratory phases.

---

# Skill Progression by Phase

## Phase 1 — Foundations

Focus:

Fundamental engineering tools and local development workflows.

Skills developed:

Linux & System Fundamentals
Git & Version Control
Basic Container Usage
Basic scripting

Examples of tasks:

- create docker containers
- run containers locally
- basic shell automation
- simple Git workflows

---

## Phase 2 — CI/CD Engineering

Focus:

Automating builds and creating reproducible pipelines.

Skills developed:

CI/CD pipelines
Container builds
Artifact generation
Pipeline debugging

Examples of tasks:

- build pipeline automation
- generate build artifacts
- push container images
- analyze pipeline logs

---

## Phase 3 — Infrastructure & Orchestration

Focus:

Managing containerized systems.

Skills developed:

Kubernetes deployments
Service exposure
Environment configuration
Container orchestration

Examples of tasks:

- deploy applications to Kubernetes
- inspect pods and logs
- configure services

---

## Phase 4 — Observability & Reliability

Focus:

Understanding system health and diagnosing issues.

Skills developed:

Monitoring
Logging
Metrics analysis
Incident debugging

Examples of tasks:

- configure monitoring tools
- inspect system logs
- diagnose failing services

---

# Skill Balance Requirement

Claude must ensure that the laboratory includes tasks covering multiple skill areas.

Example weekly balance:

Docker
CI/CD
Debugging
Automation

A sequence that teaches only one tool repeatedly is considered poor lab design.

---

# Debugging Skill Requirement

Troubleshooting must be a core part of the lab.

Students must regularly:

- inspect logs
- analyze pipeline failures
- debug container issues
- resolve deployment errors

Debugging tasks should appear regularly throughout the laboratory.

---

# Automation Growth Requirement

Automation complexity must increase progressively.

Examples:

Early lab:

manual docker commands

Mid lab:

pipeline automation

Later lab:

automated infrastructure and monitoring setup

---

# Architecture Awareness

Students should progressively learn to think about systems as a whole.

Examples:

- container architecture
- CI/CD pipeline design
- deployment architecture
- monitoring architecture

The lab should not only teach tools but also system thinking.

---

# Final Requirement

Every generated laboratory plan must reflect the following balance:

Tools
Automation
Debugging
Architecture thinking
Observability

If the plan does not cover multiple skill domains, the laboratory design must be reconsidered.
